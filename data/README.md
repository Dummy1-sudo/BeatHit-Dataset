# Data outputs

This directory separates **bootstrap evidence** from **generated final-category outputs**.

## Bootstrap / bundled material

- `raw/spotify_top10000_streamed_songs_2023.csv`: bundled historical chart-history source.
- `worldwide/worldwide_spotify_top10000_snapshot.csv`: normalized 10,000-row bootstrap snapshot. Its metric is `spotify_chart_streams_snapshot`; it is not a lifetime Spotify counter.
- `seeds/`: small manually/source-verified resilience seeds. Seeds do not imply category completeness.
- `bootstrap/megalist_bootstrap.csv`: deduplicated bootstrap union only; not the requested final megalist.

## Generated by the full build

A network-enabled run of `python scripts/build_full_dataset.py` creates:

- `anime/anime_songs.csv`
- `vocaloid/vocaloid_spotify_10m.csv`
- `worldwide/worldwide_51000.csv` plus each era bucket file
- `classical/classical_10000.csv`
- `vtuber_original/vtuber_original_10000.csv`
- `emerging/emerging_10000.csv`
- `genres/genres_10000.csv`
- `screen_soundtracks/screen_soundtracks_10000.csv`
- `vtuber_non_original/vtuber_non_original_10000.csv`
- `countries/<country-code>_<country-name>_top1000.csv`: one file per detected Spotify regional country/territory chart
- `countries/index.json`: detected markets, coverage dates, row counts, failures, and source metadata
- `megalist/megalist.csv` or, when too large for GitHub, `megalist.csv.gz` plus 50k-row CSV parts

## Do not infer completion from filenames

Read `../STATUS.json`, `coverage_report.csv`, and `target_report.json`. Fixed-size categories must hit the exact requested count. Conditional/finite corpora are never padded. Country coverage is complete only when every dynamically detected Spotify regional market has exactly 1,000 unique songs. The final megalist is only marked complete when all upstream requested lists, including country lists, are complete.
